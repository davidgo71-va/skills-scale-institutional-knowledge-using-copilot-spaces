# Cross-Functional Role Alignment Checklist

## Purpose
This checklist ensures all key roles are identified, aligned, and have clear responsibilities at project initiation. Use it to prevent gaps, reduce rework, and establish accountability across the delivery team.

## When to Use
- During project initiation and kickoff
- When scaling a project or adding new work streams
- When onboarding new team members into the project

## Responsibility Clarification

- [ ] Acceptance criteria definition owned by: **Product Manager**

- [ ] Definition of Done established and agreed by: ________________

### Technical Decisions
- [ ] Architecture decisions owned by: **Technical Lead**
- [ ] Security review process owned by: **Security Officer** (or assigned to: ________________)

### Delivery & Tracking
- [ ] Project schedule owned by: **Project Manager**

- [ ] Deployment execution owned by: **DevOps Engineer**
- [ ] Post-deployment verification owned by: **QA Lead** or **DevOps Engineer**
- [ ] Rollback authority: ________________

## Cross-Functional Alignment

- [ ] **Handoff points identified and documented:**
  - [ ] Requirements → Implementation
  - [ ] Implementation → QA/Testing
  - [ ] QA/Testing → Deployment
  - [ ] Deployment → Production Support
  
- [ ] **Dependencies mapped:**
  - [ ] Internal (within team) dependencies documented
  - [ ] External (cross-team) dependencies documented
  - [ ] Third-party/vendor dependencies documented
  
- [ ] **Collaboration norms established:**
  - [ ] Code review expectations clarified
  - [ ] Design review process defined
  - [ ] Testing approach agreed (unit, integration, e2e, manual)
  - [ ] Documentation standards agreed
